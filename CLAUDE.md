# licokit

macOS dev environment bootstrapper TUI written in Go.

## Build & Run

```bash
go build -o licokit .          # build
go run .                        # run from source
go test ./... -v                # run all tests
go test -race -cover ./...      # tests with race detection + coverage
go vet ./...                    # static analysis
gofmt -w .                      # format all files
```

Requires Go 1.23+.

## Project Structure

- `main.go` — entry point, signal handling, cursor management
- `app/` — screens (home menu, tools, dotfiles, guide)
- `lib/config/` — YAML config loading with embedded defaults + user override at `~/.config/licokit/`
- `lib/terminal/` — TUI primitives
- `lib/tools/` — tool installation and dotfiles setup
- `lib/styles/` — lipgloss style definitions
- `lib/spinner/` — braille spinner animation
- `lib/display/` — header rendering
- `dotfiles/` — config files symlinked to `~/.config/` and selected home paths

## Config System

1. User config at `~/.config/licokit/config.yaml` (takes priority)
2. Embedded default at `lib/config/default_config.yaml`

## Shell Config (Two-File Design)

- `dev.zsh` — dynamically generated by licokit during tool installation. Contains `source` lines for installed tools (nvm, zsh-vi-mode, etc.). `~/.zshrc` sources this file. Do NOT put static aliases or manual config here.
- `~/.config/zsh/zshrc` — hand-written global zsh config (prompt, aliases, env vars).

## Conventions

- Minimal dependencies: lipgloss (styling), go-tty (input), yaml.v3 (config)
- Shell out for external tools (brew, tmux, etc.) — no Go wrappers
- `ExecCommandQuiet` for commands with spinner, `ExecCommand` for interactive output
- Error handling: `fmt.Errorf("context: %w", err)`, user-facing via `tools.WarningMessage()`
- Embedded files via `//go:embed` directive
- vim-style keybindings in TUI (j/k navigation)

## Dotfiles / AI Tool Config

`dotfiles/claude/` contains optional Claude Code templates and migration assets. Dotfiles no longer symlink Claude Code configuration into `~/.claude/` by default; project-specific AI assistant setup should be applied explicitly per project.

**Tracked in git:** `CLAUDE.md`, `settings.json`, `hooks/`, `docs/`, `memory/`, `setup-plugins.sh`, `statusline-command.sh`

**NOT tracked (ECC plugin auto-generates these):** `agents/`, `commands/`, `skills/`

`migrations/2026-05-05-detach-claude-global-links.sh` temporarily removes legacy `~/.claude` symlinks that still point into this repo. It is safe to delete the migration after every managed machine has run the updated Dotfiles setup once.
